# Storage Drivers

Storage driver reference for `@classytic/media-kit`.

## StorageDriver Interface

All drivers implement this interface. Core methods are required; optional methods enable advanced features like presigned uploads, file listing, and copy/move.

```typescript
interface StorageDriver {
  readonly name: string;

  // --- Core (required) ---

  /** Write data to storage. Accepts Buffer or ReadableStream. */
  write(key: string, data: Buffer | NodeJS.ReadableStream, contentType: string): Promise<WriteResult>;

  /** Read a file as a stream. Supports optional byte-range for partial reads. */
  read(key: string, range?: { start: number; end: number }): Promise<NodeJS.ReadableStream>;

  /** Delete a file by key. */
  delete(key: string): Promise<boolean>;

  /** Check if a file exists. */
  exists(key: string): Promise<boolean>;

  /** Get file metadata without downloading. */
  stat(key: string): Promise<FileStat>;

  // --- Navigation (optional) ---

  /** List files under a prefix (async generator for memory efficiency). */
  list?(prefix: string): AsyncIterable<string>;

  /** Copy a file to a new location within the same bucket. */
  copy?(source: string, destination: string): Promise<WriteResult>;

  /** Move a file to a new location (copy + delete). */
  move?(source: string, destination: string): Promise<WriteResult>;

  // --- URLs ---

  /** Build the public URL for a given storage key. */
  getPublicUrl(key: string): string;

  /** Get a temporary signed URL for reading a private file. */
  getSignedUrl?(key: string, expiresIn?: number): Promise<string>;

  /** Get a presigned URL for direct client-side uploads. */
  getSignedUploadUrl?(key: string, contentType: string, expiresIn?: number): Promise<PresignedUploadResult>;
}
```

### Required Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `write` | `(key, data, contentType) => Promise<WriteResult>` | Write data to storage. The `key` is provided by MediaKit (drivers no longer generate keys). |
| `read` | `(key, range?) => Promise<ReadableStream>` | Read a file as a stream. The optional `range` parameter enables byte-range reads for video streaming. |
| `delete` | `(key) => Promise<boolean>` | Delete a file. Returns `true` on success. |
| `exists` | `(key) => Promise<boolean>` | Check if a file exists at the given key. |
| `stat` | `(key) => Promise<FileStat>` | Get file metadata (size, content type, last modified, etc.) without downloading. |
| `getPublicUrl` | `(key) => string` | Build the public URL for a storage key. Synchronous. |

### Optional Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `list` | `(prefix) => AsyncIterable<string>` | List file keys under a prefix. Yields keys as an async iterable for memory efficiency. |
| `copy` | `(source, destination) => Promise<WriteResult>` | Copy a file within the same bucket. |
| `move` | `(source, destination) => Promise<WriteResult>` | Move a file (copy + delete). |
| `getSignedUrl` | `(key, expiresIn?) => Promise<string>` | Generate a temporary signed URL for reading a private file. |
| `getSignedUploadUrl` | `(key, contentType, expiresIn?) => Promise<PresignedUploadResult>` | Generate a presigned URL for direct client-to-cloud uploads. |

> Both `S3Provider` and `GCSProvider` implement all optional methods.

## Return Types

### WriteResult

Returned from `write()`, `copy()`, and `move()`.

```typescript
interface WriteResult {
  key: string;   // Storage key/path
  url: string;   // Public URL
  size: number;  // File size in bytes
}
```

### FileStat

Returned from `stat()`.

```typescript
interface FileStat {
  size: number;                       // File size in bytes
  contentType: string;                // MIME type
  lastModified?: Date;                // Last modified timestamp
  etag?: string;                      // Entity tag (cache validation)
  metadata?: Record<string, string>;  // Custom metadata key-value pairs
}
```

### PresignedUploadResult

Returned from `getSignedUploadUrl()`.

```typescript
interface PresignedUploadResult {
  uploadUrl: string;                  // Presigned URL to PUT the file to
  key: string;                        // Storage key assigned to the file
  publicUrl: string;                  // Public URL after upload completes
  expiresIn: number;                  // Seconds until the URL expires
  headers?: Record<string, string>;   // Headers the client must include with PUT
}
```

## S3Provider

AWS S3 storage driver. Also works with S3-compatible services (MinIO, Cloudflare R2, DigitalOcean Spaces, Backblaze B2).

Requires `@aws-sdk/client-s3` and `@aws-sdk/s3-request-presigner` as peer dependencies. The SDK is lazy-loaded on first use.

### Import

```typescript
import { S3Provider } from '@classytic/media-kit/providers/s3';
```

### Constructor

```typescript
new S3Provider(config: S3ProviderConfig)
```

### Config

```typescript
interface S3ProviderConfig {
  /** S3 bucket name (required) */
  bucket: string;
  /** AWS region (required) */
  region: string;
  /** AWS credentials */
  credentials?: {
    accessKeyId: string;
    secretAccessKey: string;
  };
  /** Custom endpoint for S3-compatible services (MinIO, R2, etc.) */
  endpoint?: string;
  /** Custom public URL (CDN) */
  publicUrl?: string;
  /** Canned ACL for uploaded files */
  acl?: 'private' | 'public-read' | 'authenticated-read';
  /** Force path-style URLs (required for some S3-compatible services) */
  forcePathStyle?: boolean;
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `bucket` | `string` | Yes | S3 bucket name |
| `region` | `string` | Yes | AWS region |
| `credentials` | `object` | No | AWS access key credentials. Falls back to default AWS credential chain if omitted. |
| `endpoint` | `string` | No | Custom endpoint for S3-compatible services |
| `publicUrl` | `string` | No | Custom CDN URL prefix for public URLs |
| `acl` | `string` | No | Canned ACL applied to uploaded files |
| `forcePathStyle` | `boolean` | No | Use path-style URLs (`endpoint/bucket/key` instead of `bucket.endpoint/key`) |

### Features

- Automatic retry with exponential backoff on transient failures
- Lazy SDK initialization (fails gracefully with clear error message)
- Accepts both Buffer and ReadableStream in `write()`
- Byte-range reads for video streaming support
- Automatic URL extraction from full URLs or plain keys
- Async generator `list()` with S3 pagination

### Example

```typescript
import { createMedia } from '@classytic/media-kit';
import { S3Provider } from '@classytic/media-kit/providers/s3';

// AWS S3
const s3 = new S3Provider({
  bucket: 'my-uploads',
  region: 'us-east-1',
  credentials: {
    accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
    secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
  },
});

// Cloudflare R2
const r2 = new S3Provider({
  bucket: 'my-bucket',
  region: 'auto',
  endpoint: 'https://<account-id>.r2.cloudflarestorage.com',
  credentials: {
    accessKeyId: process.env.R2_ACCESS_KEY_ID!,
    secretAccessKey: process.env.R2_SECRET_ACCESS_KEY!,
  },
  forcePathStyle: true,
  publicUrl: 'https://cdn.example.com',
});

// MinIO (self-hosted)
const minio = new S3Provider({
  bucket: 'media',
  region: 'us-east-1',
  endpoint: 'http://localhost:9000',
  credentials: {
    accessKeyId: 'minioadmin',
    secretAccessKey: 'minioadmin',
  },
  forcePathStyle: true,
});

const media = createMedia({ driver: s3 });
```

### Public URL Resolution

The `getPublicUrl()` method resolves URLs in this order:

1. `publicUrl` config (if set): `${publicUrl}/${key}`
2. `endpoint` config (if set): `${endpoint}/${bucket}/${key}`
3. Default: `https://${bucket}.s3.${region}.amazonaws.com/${key}`

## GCSProvider

Google Cloud Storage driver with full feature parity with S3Provider.

Requires `@google-cloud/storage` as a peer dependency. The SDK is lazy-loaded on first use.

### Import

```typescript
import { GCSProvider } from '@classytic/media-kit/providers/gcs';
```

### Constructor

```typescript
new GCSProvider(config: GCSProviderConfig)
```

### Config

```typescript
interface GCSProviderConfig {
  /** GCS bucket name (required) */
  bucket: string;
  /** Google Cloud project ID */
  projectId?: string;
  /** Path to service account key file */
  keyFilename?: string;
  /** Service account credentials object */
  credentials?: {
    client_email: string;
    private_key: string;
  };
  /** Custom public URL (CDN) */
  publicUrl?: string;
  /** Make files publicly accessible (default: true) */
  makePublic?: boolean;
  /** Retry configuration */
  retry?: RetryOptions;
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `bucket` | `string` | Yes | GCS bucket name |
| `projectId` | `string` | No | Google Cloud project ID |
| `keyFilename` | `string` | No | Path to service account JSON key file |
| `credentials` | `object` | No | Inline service account credentials |
| `publicUrl` | `string` | No | Custom CDN URL prefix |
| `makePublic` | `boolean` | No | Make uploaded files publicly accessible (default: `true`) |
| `retry` | `RetryOptions` | No | Override retry behavior for transient failures |

### Features

- Automatic retry with exponential backoff on transient failures
- Lazy SDK initialization
- Accepts both Buffer and ReadableStream in `write()`
- Stream pipeline for ReadableStream writes
- Byte-range reads for partial content
- Async generator `list()` for memory-efficient file listing
- Automatic `makePublic()` on uploaded files (configurable)

### Example

```typescript
import { createMedia } from '@classytic/media-kit';
import { GCSProvider } from '@classytic/media-kit/providers/gcs';

// Using service account key file
const gcs = new GCSProvider({
  bucket: 'my-uploads',
  projectId: 'my-project',
  keyFilename: './service-account.json',
});

// Using inline credentials
const gcs = new GCSProvider({
  bucket: 'my-uploads',
  credentials: {
    client_email: process.env.GCS_CLIENT_EMAIL!,
    private_key: process.env.GCS_PRIVATE_KEY!,
  },
  publicUrl: 'https://cdn.example.com',
});

const media = createMedia({ driver: gcs });
```

### Public URL Resolution

1. `publicUrl` config (if set): `${publicUrl}/${key}`
2. Default: `https://storage.googleapis.com/${bucket}/${key}`

## Cloudflare R2 (via S3Provider)

R2 needs **no dedicated provider** — its S3-compatible API works with `S3Provider` plus an `endpoint`. Confirmed against the R2 docs (2026):

- **Endpoint:** `https://<ACCOUNT_ID>.r2.cloudflarestorage.com`. Jurisdiction-restricted buckets use `https://<ACCOUNT_ID>.<jurisdiction>.r2.cloudflarestorage.com` (e.g. `eu`, `fedramp`) and are reachable *only* through their jurisdiction endpoint.
- **Region:** always `'auto'` — required by the SDK, ignored by R2.
- **Path style:** R2 supports both virtual-hosted and path-style; `forcePathStyle: true` is the safe default and what the recipe below uses.
- **Presigned URLs:** supported for GET/HEAD/PUT/DELETE with expiry from 1 second to **7 days** (604 800 s) — so `getSignedUploadUrl()` / `getSignedUrl()` work as on AWS. Presigned URLs only work against the `r2.cloudflarestorage.com` S3 domain, **not** custom domains.
- **Multipart:** CreateMultipartUpload / UploadPart / CompleteMultipartUpload / AbortMultipartUpload are implemented, so `initiateMultipartUpload()` works. Note: re-uploading the same part number replaces the previous part.
- **ACLs are not supported** — R2 has no per-object ACLs. Omit `acl`; make objects public via a public bucket instead.
- **Public URLs:** buckets are private by default. For public delivery attach a **custom domain** (production — gets Cloudflare CDN/cache/WAF) or enable the `r2.dev` development URL (rate-limited, not for production). Point `publicUrl` at whichever you use.

```typescript
import { S3Provider } from '@classytic/media-kit/providers/s3';

const r2 = new S3Provider({
  bucket: 'my-bucket',
  region: 'auto',                    // required by the SDK, ignored by R2
  endpoint: `https://${process.env.CF_ACCOUNT_ID}.r2.cloudflarestorage.com`,
  credentials: {
    accessKeyId: process.env.R2_ACCESS_KEY_ID!,      // R2 API token (S3 auth kind)
    secretAccessKey: process.env.R2_SECRET_ACCESS_KEY!,
  },
  forcePathStyle: true,
  publicUrl: 'https://media.example.com', // custom domain (or https://<id>.r2.dev in dev)
});
```

Docs: [S3 API compatibility](https://developers.cloudflare.com/r2/api/s3/api/) · [Presigned URLs](https://developers.cloudflare.com/r2/api/s3/presigned-urls/) · [Public buckets](https://developers.cloudflare.com/r2/buckets/public-buckets/) · [Data location / jurisdictions](https://developers.cloudflare.com/r2/reference/data-location/)

## CloudflareImagesProvider

Cloudflare Images storage driver — managed image pipeline (storage + named/flexible variants + `imagedelivery.net` CDN). Zero dependencies: native `fetch` + `node:crypto`, no SDK.

**Images only** — max 10 MB per upload, max 100 megapixels (PNG, JPEG, GIF/WebP incl. animation, SVG, AVIF). Route videos/documents to a bucket provider with `StorageRouter`.

### Import

```typescript
import { CloudflareImagesProvider } from '@classytic/media-kit/providers/cloudflare-images';
```

### Config

```typescript
interface CloudflareImagesProviderConfig {
  /** Cloudflare account ID (required) */
  accountId: string;
  /** API token with Images:Edit — string or lazy () => string | Promise<string> */
  apiToken: string | (() => string | Promise<string>);
  /** Images account hash — appears in every delivery URL (required) */
  accountHash: string;
  /** Variant used for delivery URLs (default: 'public') */
  defaultVariant?: string;
  /** Private mode: signed-delivery token key (dashboard → Images → Keys) */
  signing?: { key: string };
}
```

### Two modes

| | Public mode (default) | Private mode (`signing` configured) |
|---|---|---|
| `write()` | Custom ID = the media-kit generated key (path-like custom IDs are supported: ≤1024 chars, subpaths OK, no leading/trailing slash, not a UUID) — the key is preserved verbatim | `requireSignedURLs: true`, **no custom ID** (Cloudflare forbids custom IDs on signed images) — the stored key is Cloudflare's UUID |
| `getPublicUrl()` | `https://imagedelivery.net/{accountHash}/{key}/{defaultVariant}` | Same shape, but 403s without a token |
| `getSignedUrl()` | Throws (no signing key) | HMAC-SHA256 `?exp=…&sig=…` token per Cloudflare's scheme |
| `getSignedUploadUrl()` | Works — direct creator upload with the generated key as custom ID, so `confirmUpload()`'s key-shape check passes | **Throws** — the CF UUID key can't survive `confirmUpload()` validation; use server-side `upload()` |

### Presigned-upload client contract (differs from S3/GCS!)

Cloudflare's one-time `uploadURL` accepts a `multipart/form-data` **POST** with the file in a `file` field — **not** a raw `PUT` of the bytes:

```typescript
// Client side — after calling media.getSignedUploadUrl(...)
const form = new FormData();
form.append('file', fileBlob);
await fetch(uploadUrl, { method: 'POST', body: form }); // NOT fetch(uploadUrl, { method: 'PUT', body: fileBlob })
```

`expiresIn` is clamped to Cloudflare's window: 2 minutes – 6 hours (default 30 min on their side; the provider always sends an explicit `expiry`).

### Caveats (platform constraints, honestly stated)

- **`stat()` size/contentType describe the served variant**, not the original bytes: the details API returns identity only (filename, upload date, signed flag), so the provider HEADs the delivery URL for byte info.
- **`read()` streams the `defaultVariant` rendition** — Cloudflare Images has no original-bytes download API. Byte-range: Range on `imagedelivery.net` is undocumented; the provider forwards the header and, if the CDN replies 200, slices the requested window locally so callers always get correct bytes.
- **No `list`/`copy`/`move`/multipart** — left `undefined` (engine feature-detects), Cloudflare Images has no equivalents (the list API has no prefix filter).
- **Disable server-side processing** (`processing: { enabled: false }`) — variants/optimization are Cloudflare's job.

### Extensions beyond StorageDriver

```typescript
const driver = engine.driver as CloudflareImagesProvider;
driver.getVariantUrl(key, 'thumbnail');            // named variant
driver.getVariantUrl(key, 'w=400,sharpen=3');      // flexible variant (must be enabled; public images only)
driver.getSignedVariantUrl(key, 'thumbnail', 600); // signed, private mode
```

### Example

```typescript
import { createMedia } from '@classytic/media-kit';
import { CloudflareImagesProvider } from '@classytic/media-kit/providers/cloudflare-images';

const media = await createMedia({
  connection: mongoose.connection,
  driver: new CloudflareImagesProvider({
    accountId: process.env.CF_ACCOUNT_ID!,
    apiToken: () => secrets.get('cf-images-token'), // lazy — resolved on first upload
    accountHash: process.env.CF_IMAGES_ACCOUNT_HASH!,
    defaultVariant: 'public',
    // signing: { key: process.env.CF_IMAGES_SIGNING_KEY! }, // ← private mode
  }),
  processing: { enabled: false },
});
```

Docs: [Upload via API](https://developers.cloudflare.com/images/upload-images/) · [Custom IDs](https://developers.cloudflare.com/images/upload-images/upload-custom-path/) · [Direct creator upload](https://developers.cloudflare.com/images/upload-images/direct-creator-upload/) · [Serve private images (tokens)](https://developers.cloudflare.com/images/manage-images/serve-images/serve-private-images/) · [Flexible variants](https://developers.cloudflare.com/images/manage-images/enable-flexible-variants/) · [Limits & formats](https://developers.cloudflare.com/images/get-started/limits/)

## Custom Driver

Implement the `StorageDriver` interface to create a custom storage backend. Only the core methods and `getPublicUrl` are required.

```typescript
import type { StorageDriver, WriteResult, FileStat, PresignedUploadResult } from '@classytic/media-kit';

class CloudinaryDriver implements StorageDriver {
  readonly name = 'cloudinary';

  private cloudinary: any;

  constructor(config: { cloudName: string; apiKey: string; apiSecret: string }) {
    // Initialize your SDK
  }

  async write(key: string, data: Buffer | NodeJS.ReadableStream, contentType: string): Promise<WriteResult> {
    // Upload to Cloudinary
    const result = await this.cloudinary.uploader.upload(data, { public_id: key });

    return {
      key,
      url: result.secure_url,
      size: result.bytes,
    };
  }

  async read(key: string, range?: { start: number; end: number }): Promise<NodeJS.ReadableStream> {
    // Return a readable stream for the file
    const url = this.getPublicUrl(key);
    const response = await fetch(url, {
      headers: range ? { Range: `bytes=${range.start}-${range.end}` } : {},
    });
    return response.body as unknown as NodeJS.ReadableStream;
  }

  async delete(key: string): Promise<boolean> {
    await this.cloudinary.uploader.destroy(key);
    return true;
  }

  async exists(key: string): Promise<boolean> {
    try {
      await this.cloudinary.api.resource(key);
      return true;
    } catch {
      return false;
    }
  }

  async stat(key: string): Promise<FileStat> {
    const resource = await this.cloudinary.api.resource(key);
    return {
      size: resource.bytes,
      contentType: `${resource.resource_type}/${resource.format}`,
      lastModified: new Date(resource.created_at),
    };
  }

  getPublicUrl(key: string): string {
    return `https://res.cloudinary.com/${this.config.cloudName}/image/upload/${key}`;
  }

  // Optional: implement list, copy, move, getSignedUrl, getSignedUploadUrl
  // for full feature support
}
```

### Using a custom driver

```typescript
import { createMedia } from '@classytic/media-kit';

const media = createMedia({
  driver: new CloudinaryDriver({
    cloudName: 'my-cloud',
    apiKey: process.env.CLOUDINARY_API_KEY!,
    apiSecret: process.env.CLOUDINARY_API_SECRET!,
  }),
});
```

### Notes on custom drivers

- **Key generation is handled by MediaKit.** The driver's `write()` method receives the final storage key -- it does not need to generate one.
- **`getPublicUrl` must be synchronous.** It is called frequently during upload to build URLs for the database record.
- **Implement `getSignedUploadUrl` to enable presigned uploads.** Without it, `media.getSignedUploadUrl()` will throw.
- **`list` should be an async generator** for memory-efficient iteration over large prefixes.
- **`read` should support byte-range** for video streaming (HTTP 206 partial content).
