# Uploading Files

Learn different ways to upload files with Media Kit.

## Single File Upload

```typescript
const file = await media.upload({
  buffer: fileBuffer,
  filename: 'photo.jpg',
  mimeType: 'image/jpeg',
  folder: 'images',
});

console.log(file.url);       // CDN URL
console.log(file.key);       // Storage key
console.log(file.size);      // File size in bytes
console.log(file.status);    // 'ready'
console.log(file.hash);      // SHA-256 content hash
```

## With Metadata

Add tags, description, and focal point:

```typescript
const file = await media.upload({
  buffer: fileBuffer,
  filename: 'product.jpg',
  mimeType: 'image/jpeg',

  // Display fields
  title: 'Red Running Shoes',
  alt: 'Red sneakers on white background',
  description: 'New arrival - 2024 collection',

  // Organization
  folder: 'products',
  tags: ['shoes', 'new-arrival', 'red'],

  // Smart cropping anchor
  focalPoint: { x: 0.5, y: 0.3 },
});
```

## Multiple Files

Upload several files at once:

```typescript
const files = await media.uploadMany([
  { buffer: buffer1, filename: 'photo1.jpg', mimeType: 'image/jpeg' },
  { buffer: buffer2, filename: 'photo2.jpg', mimeType: 'image/jpeg' },
  { buffer: buffer3, filename: 'photo3.jpg', mimeType: 'image/jpeg' },
]);

files.forEach(f => console.log(f.url));
```

## From Express/Multer

```typescript
import multer from 'multer';

const upload = multer();

app.post('/upload', upload.single('file'), async (req, res) => {
  const file = await media.upload({
    buffer: req.file.buffer,
    filename: req.file.originalname,
    mimeType: req.file.mimetype,
    tags: req.body.tags?.split(',') || [],
    description: req.body.description,
  });

  res.json({
    id: file._id,
    url: file.url,
    status: file.status,
  });
});
```

## From Next.js API Route

```typescript
// app/api/upload/route.ts
import { NextRequest, NextResponse } from 'next/server';

export async function POST(req: NextRequest) {
  const formData = await req.formData();
  const file = formData.get('file') as File;
  const tags = formData.get('tags') as string;

  const buffer = Buffer.from(await file.arrayBuffer());

  const uploaded = await media.upload({
    buffer,
    filename: file.name,
    mimeType: file.type,
    tags: tags?.split(',') || [],
  });

  return NextResponse.json({
    url: uploaded.url,
    status: uploaded.status,
  });
}
```

## File Replacement

Replace the content of an existing file while keeping the same ID and URL:

```typescript
const updated = await media.replace(existingFileId, {
  buffer: newBuffer,
  filename: 'product-v2.jpg',
  mimeType: 'image/jpeg',
});

// Same _id, updated content
console.log(updated._id);    // Same as before
console.log(updated.hash);   // New hash
console.log(updated.status); // 'ready'
```

## Import from URL

Import a file directly from a URL:

```typescript
const file = await media.importFromUrl(
  'https://example.com/image.jpg',
  {
    folder: 'imports',
    title: 'Imported Image',
    tags: ['imported', 'external'],
  }
);

console.log(file.url);              // Your CDN URL
console.log(file.originalFilename); // 'image.jpg'
```

## Soft Delete and Restore

When soft deletes are enabled, files are marked as deleted rather than permanently removed:

```typescript
// Soft delete (marks deletedAt, excluded from queries)
const trashed = await media.softDelete(fileId);
console.log(trashed.deletedAt); // Date

// Restore from trash
const restored = await media.restore(fileId);
console.log(restored.deletedAt); // null

// Permanently remove expired soft-deleted files
const purgedCount = await media.purgeDeleted();
```

## Tags

Manage tags on existing files:

```typescript
// Add tags
await media.addTags(fileId, ['featured', 'hero']);

// Remove tags
await media.removeTags(fileId, ['draft']);
```

## Focal Point

Set a focal point for smart cropping:

```typescript
await media.setFocalPoint(fileId, { x: 0.3, y: 0.2 });
```

The focal point is used when generating cropped variants to keep the subject in frame. Values range from `0.0` to `1.0` (normalized coordinates).

## With Context

Pass request context (user, organization):

```typescript
const file = await media.upload(
  {
    buffer: fileBuffer,
    filename: 'document.pdf',
    mimeType: 'application/pdf',
  },
  {
    userId: req.user.id,
    organizationId: req.user.organizationId,
  }
);
```

## Multi-Tenant Upload

Upload files scoped to an organization:

```typescript
const file = await media.upload(
  {
    buffer: fileBuffer,
    filename: 'logo.png',
    mimeType: 'image/png',
  },
  {
    organizationId: req.user.organizationId,
  }
);

// File is isolated to this organization
```

## Auto Alt Text

Media Kit auto-generates alt text from filenames:

```typescript
const file = await media.upload({
  buffer: fileBuffer,
  filename: 'red-running-shoes-2024.jpg',
  mimeType: 'image/jpeg',
});

console.log(file.alt);  // "Red running shoes 2024"
```

Override auto-generated alt:

```typescript
const file = await media.upload({
  buffer: fileBuffer,
  filename: 'IMG_1234.jpg',
  mimeType: 'image/jpeg',
  alt: 'Product photo in studio lighting',
});
```

## Upload Result

A successful upload returns an `IMediaDocument` with these fields:

```typescript
{
  _id: ObjectId,
  filename: string,          // Sanitized storage filename
  originalFilename: string,  // Original name as uploaded
  title: string,             // Display title
  key: string,               // Storage key/path
  url: string,               // Public URL
  mimeType: string,
  size: number,              // Bytes
  hash: string,              // SHA-256 content hash
  status: 'ready',           // Status lifecycle
  folder: string,
  tags: string[],
  alt?: string,
  description?: string,
  width?: number,            // Image only
  height?: number,           // Image only
  focalPoint?: { x, y },
  variants: [],              // Generated size variants
  deletedAt: null,
  createdAt: Date,
  updatedAt: Date,
}
```
