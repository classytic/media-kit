# Image Processing

Auto-resize, optimize, and generate multiple image sizes with focal-point-aware cropping.

## Setup

Install Sharp for image processing:

```bash
npm install sharp
```

Enable processing in config:

```typescript
const media = createMedia({
  driver: myDriver,
  processing: {
    enabled: true,
    quality: 80,
    format: 'webp',
  },
});
```

## Status Lifecycle

Every upload goes through a status lifecycle:

```
pending -> processing -> ready
                      -> error
```

- **pending** -- file received, not yet processed
- **processing** -- image variants being generated
- **ready** -- all processing complete, file is available
- **error** -- processing failed (original file is still stored)

```typescript
const file = await media.upload({
  buffer: imageBuffer,
  filename: 'photo.jpg',
  mimeType: 'image/jpeg',
});

console.log(file.status); // 'ready' (processing completes during upload)
```

If processing fails, the status is set to `error` and the `errorMessage` field contains details. The original unprocessed file is still available at `file.url`.

## Basic Processing

Images are automatically processed on upload:

```typescript
const file = await media.upload({
  buffer: largeImageBuffer,
  filename: 'photo.jpg',
  mimeType: 'image/jpeg',
});

// Automatically:
// - Resized to max width (default: 2048px)
// - Converted to WebP
// - Quality optimized
// - SHA-256 hash computed
```

## Generate Multiple Sizes

Create thumbnail, medium, and large versions:

```typescript
const media = createMedia({
  driver: myDriver,
  processing: {
    enabled: true,
    sizes: [
      { name: 'thumbnail', width: 150, height: 150 },
      { name: 'medium', width: 800 },
      { name: 'large', width: 1600 },
    ],
  },
});

const file = await media.upload({
  buffer: imageBuffer,
  filename: 'product.jpg',
  mimeType: 'image/jpeg',
});

// Access variants
console.log(file.url);                  // Original/processed
console.log(file.variants);            // Array of generated sizes
console.log(file.variants[0].url);     // Thumbnail URL
console.log(file.variants[0].width);   // Thumbnail width
console.log(file.variants[0].mimeType); // Each variant has its own mimeType
```

## Conditional Variants

Skip variant generation when the source image is too small. Use the `condition` function:

```typescript
processing: {
  sizes: [
    // Always generated
    { name: 'thumbnail', width: 200, height: 200 },

    // Only generated if source is at least 800px wide
    {
      name: 'medium',
      width: 800,
      condition: (original) => original.width >= 800,
    },

    // Only generated for landscape images wider than 1600px
    {
      name: 'hero',
      width: 1920,
      condition: (original) =>
        original.width >= 1920 && original.width > original.height,
    },

    // Only generated for JPEG/PNG sources (skip SVGs, GIFs)
    {
      name: 'webp-alt',
      width: 1200,
      format: 'webp',
      condition: (original) =>
        ['image/jpeg', 'image/png'].includes(original.mimeType),
    },
  ],
}
```

The `condition` function receives `{ width, height, mimeType }` of the original image. If it returns `false`, that variant is skipped.

## Focal Point Cropping

Set a focal point to control where crops are centered. This ensures the subject stays visible across all variant sizes:

```typescript
// Set focal point during upload
const file = await media.upload({
  buffer: imageBuffer,
  filename: 'portrait.jpg',
  mimeType: 'image/jpeg',
  focalPoint: { x: 0.5, y: 0.3 }, // Center-top focus
});

// Or update it later
await media.setFocalPoint(file._id, { x: 0.3, y: 0.2 });
```

Focal point coordinates are normalized from `0.0` to `1.0`:
- `x: 0.0` = left edge, `x: 1.0` = right edge
- `y: 0.0` = top edge, `y: 1.0` = bottom edge
- `{ x: 0.5, y: 0.5 }` = center (default)

When variants with aspect ratio cropping are generated, the focal point is used as the crop anchor instead of the default center.

## Size Options

### Fixed Width

```typescript
{ name: 'medium', width: 800 }
// Maintains aspect ratio, scales to 800px wide
```

### Fixed Dimensions

```typescript
{ name: 'square', width: 400, height: 400 }
// Crops/fits to exact 400x400
```

### Quality Control

```typescript
{ name: 'compressed', width: 1200, quality: 60 }
// Lower quality for faster loading
```

### Format Override

```typescript
{ name: 'thumbnail', width: 200, format: 'jpeg' }
// Force JPEG instead of default WebP
```

Supported formats: `'webp'`, `'jpeg'`, `'png'`, `'avif'`, `'original'` (keep source format).

## Aspect Ratios

Maintain specific aspect ratios:

```typescript
processing: {
  sizes: [
    {
      name: 'instagram',
      width: 1080,
      aspectRatio: {
        aspectRatio: 1,   // 1:1 square
        fit: 'cover',
      },
    },
    {
      name: 'banner',
      width: 1200,
      aspectRatio: {
        aspectRatio: 16 / 9,
        fit: 'contain',
      },
    },
    {
      name: 'portrait',
      width: 800,
      aspectRatio: {
        aspectRatio: 3 / 4,
        fit: 'cover',
      },
    },
  ],
}
```

Fit modes:
- `cover` -- fill the area, cropping if needed (respects focal point)
- `contain` -- fit within the area, adding letterboxing if needed
- `fill` -- stretch to fill exactly
- `inside` -- resize to fit inside, never upscaling
- `outside` -- resize to cover, possibly exceeding dimensions

## Variant Metadata

Each generated variant stores independent metadata (Payload CMS pattern):

```typescript
interface GeneratedVariant {
  name: string;     // e.g., 'thumbnail'
  key: string;      // Storage key
  url: string;      // Public URL
  filename: string; // Can differ from original
  mimeType: string; // Can differ (e.g., WebP thumbnail for JPEG original)
  size: number;     // Bytes
  width?: number;
  height?: number;
}
```

## Complete Example

```typescript
const media = createMedia({
  driver: new S3Provider({ bucket: 'images', region: 'us-east-1' }),
  processing: {
    enabled: true,
    quality: 85,
    format: 'webp',
    maxWidth: 2048,

    sizes: [
      // Thumbnail - square crop, always generated
      {
        name: 'thumb',
        width: 200,
        height: 200,
        quality: 75,
        aspectRatio: { aspectRatio: 1, fit: 'cover' },
      },

      // Small - for mobile
      {
        name: 'small',
        width: 480,
        quality: 80,
      },

      // Medium - for tablets
      {
        name: 'medium',
        width: 800,
        quality: 85,
        condition: (original) => original.width >= 800,
      },

      // Large - for desktop, only if source is big enough
      {
        name: 'large',
        width: 1600,
        quality: 90,
        condition: (original) => original.width >= 1200,
      },
    ],
  },
});

// Upload with focal point
const file = await media.upload({
  buffer: imageBuffer,
  filename: 'hero-image.jpg',
  mimeType: 'image/jpeg',
  focalPoint: { x: 0.5, y: 0.3 },
  tags: ['hero', 'homepage'],
});

// Use in HTML
const variants = file.variants;
const thumb = variants.find(v => v.name === 'thumb');
const medium = variants.find(v => v.name === 'medium');
const large = variants.find(v => v.name === 'large');

// <picture>
//   <source media="(min-width: 1200px)" srcset={large?.url} />
//   <source media="(min-width: 768px)" srcset={medium?.url} />
//   <img src={thumb?.url} alt={file.alt} />
// </picture>
```

## Responsive Images

Use variants for responsive images:

```typescript
function ProductImage({ file }) {
  const { variants } = file;

  return (
    <picture>
      <source
        media="(min-width: 1024px)"
        srcSet={variants.find(v => v.name === 'large')?.url}
      />
      <source
        media="(min-width: 768px)"
        srcSet={variants.find(v => v.name === 'medium')?.url}
      />
      <img
        src={variants.find(v => v.name === 'thumb')?.url}
        alt={file.alt}
        width={variants.find(v => v.name === 'thumb')?.width}
        height={variants.find(v => v.name === 'thumb')?.height}
      />
    </picture>
  );
}
```

## Without Sharp

If Sharp is not installed, images upload as-is without processing:

```typescript
const media = createMedia({
  driver: myDriver,
  processing: { enabled: false },  // No processing
  suppressWarnings: true,          // Hide Sharp warning
});
```
